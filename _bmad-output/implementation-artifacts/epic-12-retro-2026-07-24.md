## Executive Summary

Epic 12 shipped from a full-app usability audit (`docs/usability-audit-2026-07-13.md`) grounded in
live-browser navigation of a bootstrapped local Docker stack, cross-checked by independent UX-expert
and accessibility-expert subagent reviews plus product-owner findings. It split into two independent
stories: `12-1-project-information-architecture` (a real `/projects/:id` overview page, a persistent
project sub-nav across 9 screens, dashboard/list re-linking, and one new API endpoint) and
`12-2-usability-trust-accessibility-fixes` (dashboard-staleness trust fix, onboarding escape hatch,
sealed-vault explanations, accessibility fixes, danger-zone grouping, and naming/content consistency).

Both stories closed with zero critical/high adversarial-review findings; `12-2` achieved full TDD
red-green across all 26 ACs and a genuine full `make ci` pass; `12-1` caught and fixed a real bug
(stale `openapi.json`) via its targeted-test CI gate. Both are live in production and accepted.

**This retro's real finding isn't in the shipped code — it's in the process around the epic.** Two
patterns recurred that this project's own retros have flagged before: (1) Epic 14 started and
completed a story before Epic 12's retro ran at all, a fourth recurrence of P6-2/P7-2/P8-3, this time
exposed as a genuine scope bug in the `pick-story` guard built specifically to prevent it; and (2) ten
concrete deferred items and review findings from `12-1`/`12-2` were never entered into
`deferred-work.md`, a fifth recurrence of the P5-2/P5-3 pattern. Both were fixed directly during this
retro session rather than left as another repeated action item — see Action Items below.

## Gaps, Inconsistencies & Risks Audit

### 1. [Critical] The epic-retro-before-next-epic guard has a loophole that let Epic 14 start with Epic 12's retro never run — the exact pattern it was built to stop

`sprint-status.yaml` had no `epic-12-retrospective` key at all (unlike epics 6-11 and 13-16, all of
which have one, even `optional`) — its tracking was never registered when Epic 12 was created. Epic
14 is `in-progress` with story `14-0` `done` and `14-1` `ready-for-dev`. The `pick-story` A0 guard
(`~/.claude/skills/pick-story/SKILL.md`, prior to this session's fix) only checked
`epic-{{N-1}}-retrospective` — for Epic 14 that's Epic 13, which has zero `done` stories, so the
guard's own trigger condition ("epic N-1 has any done stories") never fired, and it never looked back
further to Epic 12, which is numerically N-2 but is the actual most-recent epic with shipped,
un-retro'd work. This is a live recurrence of P6-2/P7-2/P8-3 — the guard built specifically to close
this gap had a scope bug that let it happen again anyway, one epic later.

**Resolution:** Fixed this session — `pick-story`'s A0 step now scans every prior epic for the
highest-numbered one with completed work and a non-`done` retrospective, not just `N-1`. See
`~/.claude/skills/pick-story/SKILL.md`. `epic-12-retrospective` registered and set to `done` as part
of this retro. **[REPEAT 4x — Epic 6 retro P6-2, Epic 7 retro P7-2, Epic 8 retro P8-3, now recurring
live in Epic 12/14, fixed 2026-07-24.]**

### 2. [High] Six audit-flagged UI/UX deferrals from Epic 12's own stories were never entered in `deferred-work.md` — repeats the P5-2/P5-3 pattern

`12-2-usability-trust-accessibility-fixes.md`'s own "Out of scope for this story" section names six
concrete, audit-sourced deferrals: Notification Preferences table redesign, "Reveal value" auto-hide,
cron-field help/presets, login-page tagline styling, the persistent MFA banner not being a real

dormancy-threshold caveat visibility — plus a note that platform-operator-only screens were never

closed for themselves and Epic 11's retro reaffirmed as a team agreement.

**Resolution:** Backfilled into `deferred-work.md` this session (all six items, plus the four items
from finding #3 below). Added a new Definition-of-Done checklist item (P4) to `bmad-dev-story`'s
checklist requiring deferred items be logged before a story reaches `review`, so this no longer
depends on someone remembering. **[REPEAT 4x — Epic 5, 6, 7, 8 retros all name this exact pattern
(P5-2/P5-3); Epic 11's retro reaffirmed the team agreement; Epic 12 is the 5th recurrence, fixed
2026-07-24.]**

## Key Takeaways

1. **Epic 12 delivered real MVP-quality trust/usability improvements**, grounded in genuine pre-work
   (live audit + independent expert reviews) — the technical execution was clean.
2. **The recurring failure isn't technical, it's structural**: findings and deferrals get written down
   as prose (in stories, in skill files) but nothing forces them into a system that outlives the
   document — this is the third distinct instance of that shape in this project (retro-before-next-epic,
   deferred-work tracking, and now a bug in the very guard built to enforce the first one).
3. **A guard that "exists" isn't the same as a guard that works** — the fix this retro made isn't just
   "run the ceremony," it's "make the two recurring gaps structurally harder to repeat a sixth time."
