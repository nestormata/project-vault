## Executive Summary

Epic 1 delivered a **strong platform foundation**: Turborepo quality gates, Docker deployment, PostgreSQL RLS, vault crypto, full auth/MFA/session stack, SecureRoute framework, structured logging, and failed-auth detection (delivery stubbed to Epic 3).

**Nestor (Project Lead) key insights:**

1. **Strong base work** — Epic 2 built successfully on this foundation; adversarial reviews are what made it solid, not a drag.
2. **MFA decision fragmentation** — implementation is correct; the mental model across enrollment, policy, and login surfaces was never consolidated. **Decision: Option A** — document current behavior (ADR-1.9-05) and update AC-E1c in epics to match; do not add strict login block for unenrolled post-grace users.
3. **Deploying pain** — operators face too many manual steps and role/env confusion (`postgres` vs `vault_app`, vault init/unseal ceremony, env var surface, turbo passthrough).

**Epic 1 verdict:** **Done** — platform-ready for Epic 2+ feature work. Remaining gaps are **documentation/operator UX** (deployment runbook, MFA matrix) and **Epic 3 notification wiring** for alert stubs planted in 1.9/1.10.

**Epic update required before Epic 3?** No fundamental architecture change. **Process + operator docs required.**

## What Went Well

1. **Quality gate suite from day one (1.1)** — CI, Docker smoke, mutation testing, Trivy, jscpd, env parity, pre-commit hooks. Every subsequent story inherited a real baseline.

2. **Adversarial review culture** — Nestor confirmed: constant reviews are what gets the team closer. Story 1.1 alone closed 20+ deployment/security findings (non-root containers, loopback Postgres, `.dockerignore`, Dependabot). Pattern repeated across auth/crypto/RLS stories.

3. **Two-file migration pattern (1.4)** — Drizzle DDL + manual RLS SQL became the standard; early resolution of `org_memberships` / `audit_log_entries` naming prevented rename migrations.

4. **Vault crypto foundation (1.5)** — Init/unseal, envelope key splitting, Argon2 KDF; Epic 2 credential encryption built directly on this.

5. **SecureRoute capstone (1.11)** — Closed the `route-audit.test.ts` TODO from Story 1.1; RLS-in-transaction ADRs; enforceable pattern for all protected routes.

6. **Explicit cross-story deferrals worked** — 1.6→1.9→1.12 MFA chain; 1.9→1.12 failed-auth wiring; 1.8/1.9→Epic 3 alert delivery. Handoffs documented in story files and largely honored.

7. **Epic 2 validation** — Full credential epic shipped on this base without re-architecting auth, RLS, or deployment.

### Deploying Pain (Project Lead)

Operators and evaluators hit friction across these surfaces:

| Pain point | Source | Symptom |
|------------|--------|---------|
| **Dual DB role model** | 1.4 migrations | `postgres` for migrate only; `vault_app` for app/tests — easy to connect as superuser and get false-green RLS tests |
| **Vault init/unseal ceremony** | 1.5 + README | Multi-step: migrate → export env → init vault → unseal → register; bootstrap token + passphrase modes; turbo env passthrough for operator vars |
| **Env var surface** | 1.1, 1.5, 1.7, 1.8, 1.12 | Strict Zod startup; many auth/MFA/vault secrets; production vs dev placeholder rules |
| **Local dev vs Docker dev** | 1.1 review | Dev compose vs production image deps caused startup bugs (fixed, but left scar tissue) |
| **Migrate service cold start** | 1.4 deferred | `docker compose up` rebuilds full API image for one-shot migrate — slow first boot |
| **No single operator runbook** | Epic 9.5 backlog | README has pieces; no consolidated "zero to eval-ready" script or runbook |
| **Argon2 event-loop block** | 1.5 accepted risk | Init/unseal ~1s blocks Node event loop — surprising in Docker health timing |

- **Story 1.11** — SecureRoute + route-audit enforcement; audit-write rollback ADR.
- **Story 1.12** — Completed MFA login challenge; wired Story 1.9 deferred `recordFailedAuthAttempt` at verify-login.

## Significant Discoveries

### 1. Reviews are the quality engine — keep them

Six of twelve stories required adversarial review patches. Nestor explicitly validated this as the mechanism that produces a strong base. **Do not reduce review depth on auth/crypto/RLS/deployment stories.**

### 2. Platform epics need operator UX stories

Epic 1 delivered correct platform behavior but spread operator knowledge across README sections, story Dev Notes, and tribal env export steps. **Deploying pain is a product gap**, not just docs polish — consider Epic 9.5 runbook or an interim bootstrap story.

### 3. Security decisions need a decision map

MFA is the exemplar: three stories, two ADRs, one epics AC mismatch. Future security epics should ship a **policy matrix** alongside code, not as retro debt.
